# Этап 63. Динамическое подключение языков

## Общая схема

Программа получает список доступных языков из INI файлов в папке `language`.

Источником идентификатора является поле `LanguageCode` внутри секции `[LanguageFileInformation]`. Имя файла не используется для определения языка.

Английский язык `en-US` используется по умолчанию, пока программа не прочитала сохранённую настройку. После загрузки настроек применяется выбранный пользователем `LanguageCode`.

## Как добавить язык

Нужно создать новый INI файл в папке `language` и заполнить метаданные:

```ini
[LanguageFileInformation]
LanguageAuthor=Translator name
LanguageCode=de-DE
LanguageName=Deutsch
```

В новом файле должны присутствовать все секции и ключи из `en-US.ini`. Переводятся только значения справа от знака `=`.

После добавления файла программу нужно перезапустить. При следующем запуске язык будет обнаружен автоматически и появится в начальном мастере и в обычных настройках.

## Как сохраняется язык

Начальный мастер и обычное окно настроек сохраняют только строковое значение:

```ini
[Personalization]
LanguageCode="en-US"
```

При хранении настроек в реестре используется строковое значение `LanguageCode` в разделе `Personalization`.

Числового параметра `Language` в системе локализации нет.

## Отдельные программы

Просмотрщик журнала, редактор сценариев, дизайнер тем, PE Helper и обновлятор используют тот же принцип `LanguageCode`. Все проекты подключают один общий исходный файл `Utilities\Language\LocalizationService.vb`. Отдельных копий сервиса локализации в утилитах нет.
Каждая утилита использует собственные разделы, например `DynaViewer.Designer.Main`, `StarterScript.Designer.Main`, `ThemeDesigner.Designer.Main`, `Updater.Designer.Main` и `PEHelper.Designer.Main`. Общий раздел `Designer.Main` остался только у главного окна DISMTools.

Основная программа передаёт текущий код языка при запуске утилиты. При самостоятельном запуске утилита читает сохранённый `LanguageCode`. Если значение отсутствует, используется `en-US`.

## PE Helper

В создаваемый установочный ISO не копируется полный файл локализации DISMTools.

Для `autorun.exe` создаётся один минимальный INI выбранного языка. В него входят метаданные языка и только конкретные ключи, которые реально вызываются кодом PE Helper. На текущем этапе это 74 ключа в 12 разделах. Полные разделы и остальные строки DISMTools в ISO не копируются. Проверка локализации сравнивает список экспортируемых ключей с фактическими вызовами в исходном коде PE Helper.

При запуске HotInstall PE Helper передаёт ему текущий `LanguageCode`. HotInstall является отдельным компонентом со своими языковыми файлами. Он динамически сканирует собственную папку `Languages` и читает `LanguageCode` из метаданных каждого INI. В установочный носитель входят только реально поддерживаемые им переводы. Сейчас это английский, испанский, французский, итальянский и португальский. Если для выбранного языка собственного перевода HotInstall нет, он использует английский `en-US`. Файлы локализации основной программы в HotInstall не копируются.

## Установщик и деинсталлятор

Установщик и деинсталлятор используют только файлы `.isl` из `Installer\Languages`. INI файлы приложения не участвуют в локализации установщика.

## Проверка полноты

Fallback на другой язык не используется для отсутствующих ключей. Если нужного ключа нет, программа сообщает язык, секцию и имя ключа.
