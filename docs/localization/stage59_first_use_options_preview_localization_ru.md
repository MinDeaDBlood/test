# Stage 59. Локализация превью первого запуска и персонализации

Этот этап закрывает проблему, когда после смены языка на экране первого запуска и в окне Options, Personalization, часть демонстрационного текста оставалась на английском.

## Что исправлено

Демонстрационный журнал теперь берётся из INI файлов, а не остаётся английской строкой из дизайнера.

Используются секции:

```ini
[PrgSetup.LogPreview]
Packages.Add.Message=...

[Options.LogPreview]
Packages.Add.Message=...
```

Превью вторичного прогресса тоже локализуется:

```ini
[PrgSetup.ProgressPreview]
Wait.Label=...
ImageIndexes.Message=...

[Options.ProgressPreview]
Wait.Label=...
ImageIndexes.Message=...
```

## Поведение

На первом экране настройки при смене языка вызывается `ApplyLocalizedText()`. Он обновляет обычные подписи, список языков, демонстрационный журнал и изображение превью прогресса.

В Options, Personalization при смене языка также вызывается `ApplyLocalizedText()`. Он обновляет демонстрационный журнал и заново рендерит превью вторичного прогресса.

## Что оставлено без перевода

Технический путь `C:\w100-glb` оставлен как пример пути. Он не переводится.

Названия технических форматов и компонентов в других местах проекта по прежнему не переводятся без необходимости.

## Проверка

Проверены все основные языки:

```text
en-US
es-ES
fr-FR
it-IT
pt-PT
```

Секции и ключи одинаковые во всех языках. Значения переведены отдельно для каждого языка.
